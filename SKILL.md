# 悬疑小说创作系统 — Thriller Craft

> 融合 25+ 专业编剧方法论的 AI 悬疑创作技能包
> 理论来源：McKee《故事》、Syd Field 三幕式、Egri 前提法、Chandler 场景动力学、Hitchcock 悬念理论、Knox 十诫、Christie 嫌疑人设计、东野圭吾动机驱动

---

## 系统定位

本技能覆盖悬疑/推理/惊悚类型从构思到成稿的全流程，支持 9 大子类型：

| 子类型 | 工作流路线 | 结构重心 | 核心技术 |
|--------|-----------|---------|---------|
| 本格推理 | 诡计优先 | 第二幕(线索收集60%) | 公平线索、Knox检查 |
| 社会派 | 角色优先 | 均匀分布 | 动机从个人升级到体制 |
| 心理悬疑 | 角色优先 | 第三幕(认知翻转) | 不可靠叙述者、信任弧 |
| 惊悚悬疑 | 结构优先 | 第二幕下半(追逐) | 对抗式计划-失败 |
| 密室推理 | 诡计优先 | 第一幕(空间)+第三幕(机制) | 空间谜题、图示 |
| 叙述性诡计 | 诡计优先 | 第三幕(叙事颠覆) | 双重阅读设计 |
| 犯罪程序 | 结构优先 | 均匀、程序真实感 | 法律/法医准确性 |
| 倒叙推理 | 任意 | 第一幕(展示犯罪)+二三幕(侦破) | 猫鼠张力 |
| 日常推理 | 角色优先 | 60%社区/40%推理 | 业余侦探、镜头外犯罪 |

## 创作流程（6个阶段）

用户说"开始创作"或类似意图时，按以下流程引导。每个阶段生成独立文件，作为后续阶段的输入。

### 阶段 1：项目初始化

收集以下参数（通过对话或 AskUserQuestion）：

| 参数 | 说明 | 选项 |
|------|------|------|
| 子类型 | 悬疑子类型 | 9种见上表 |
| 篇幅 | 目标字数 | 短篇(≤5万) / 中篇(10万) / 长篇(25万+) |
| 叙事视角 | 视角选择 | 第一人称 / 第三人称有限 / 多视角交替 |
| 核心诡计类型 | 谜团本质 | 密室/不在场证明/身份替换/叙述陷阱/动机反转/连环杀人 |
| 风格调性 | 整体氛围 | 冷硬写实/哥特暗黑/日系清冷/社会批判/黑色幽默/文学沉郁 |
| 主题前提 | 一句话主题 | 例："失控的正义比犯罪更危险" |
| 社会场域 | 封闭/半封闭系统 | 家族企业/学校/医院/警局/剧团/小镇 |
| 关系引擎 | 人物互动主驱动力 | 群像博弈/双主角镜像/猫鼠对抗/多方拿捏 |

短篇模式(≤5万字)：阶段1与阶段2合并执行。

输出：生成 `project-init.md` 项目概要文件。

### 阶段 2：主题前提与立意

> "没有前提的悬疑只是谜题游戏；有前提的悬疑才是文学。" — Lajos Egri

输出：生成 `theme.md`，包含：

```
一、主题前提（一句话）
   格式："{某种人性/社会现象} 导致 {某种结果}"
   前提必须可辩论、可论证。犯罪本身就是主题的极端表达。

二、主题在故事中的体现
   - 犯罪如何体现主题？
   - 侦探弧光如何回应主题？
   - 揭示如何升华主题？

三、三维验证（Egri方法）
   - 生理维度：主题在角色物理世界中的体现
   - 社会维度：主题如何反映社会结构/权力关系
   - 心理维度：主题如何触及角色内在冲突

四、主题与子类型的适配方式
```

### 阶段 3：核心诡计与线索架构

> 先确定揭示的那一刻：真相是什么？读者会感受到什么？从终点倒推起点。

加载参考：[clue-design.md](references/clue-design.md)

输出：生成 `trick-design.md`，包含：

```
一、核心诡计概述
   - 谜面（读者看到的表象）
   - 谜底（真实发生的事情）
   - 诡计机制（表象如何形成）

二、公平性验证（Knox十诫适用项检查）

三、线索布局表
   | 线索ID | 类型 | 出现章节 | 呈现方式 | 读者注意力预测 | 揭示章节 |
   关键线索 5-8条，红鲱鱼 2-4条

四、红鲱鱼生命周期
   每条红鲱鱼：埋设→培育→强化→偏转→消解

五、反派计划骨架（James Frey反派引擎法）
   - 反派完整计划（分步骤）
   - 每步留下的痕迹 → 痕迹 = 线索
   - 侦探如何发现每条痕迹 → 发现过程 = 情节
   - 反派如何反击 → 反击 = 冲突升级

六、知识状态矩阵
   - 角色知识矩阵：每角色每章知道什么
   - 读者知识层：确知/推测/误信/不知
   - 双读者验证：天真读者(惊奇) + 侦探读者(原来如此)
```

**线索设计核心原则**：解答唯一性 / 线索公平性(至少3条关键线索在揭示前已呈现) / 误导合理性 / 逻辑闭环

线索布设四法：功能遮蔽、情感遮蔽、列举遮蔽、时间遮蔽（详见 [clue-design.md](references/clue-design.md)）

### 阶段 4：角色系统设计

加载参考：[character-system.md](references/character-system.md)

输出：生成 `characters.md`，包含：

```
一、侦探/主角
   - 人物档案 + 核心缺陷 + 侦探方法论
   - 角色罗盘（David Corbett 4D模型）：
     Lack(匮乏) → Yearning(渴望) → Resistance(抗拒) → Desire(欲望)
   - Watson角色 + 个人Stakes（至少两层叠加）

二、犯人/反派
   - 第一印象设计 vs 真实身份
   - 犯罪能力来源 + 犯人的自洽叙事 + 暴露轨迹
   - 同理性等级（Level 1-5，目标Level 3+）

三、嫌疑人矩阵
   | 角色 | 动机 | 机会 | 能力 | 不在场证明 | 嫌疑指数曲线 |

四、受害者（N个面孔 + 至少1个秘密）

五、关系网络图
   - 显性/隐性关系、资源流、知识不对称、关系翻面触发器

六、群像引擎
   | 角色 | 面具人格 | 真实意图 | 筹码 | 依赖 | 不能暴露的秘密 | 翻面条件 |
```

**角色核心原则**：压力下的选择揭示人物 / 角色是策略体(面具+真实意图) / 关系是可变边 / 侦探与犯人有同源的 Lack

### 阶段 5：叙事结构设计

加载参考：[mystery-structure.md](references/mystery-structure.md)

输出：生成 `structure.md`，包含：

```
一、结构选择与论证

二、三幕式详细节拍
   第一幕·建立（25%）→ 第二幕上·调查（25%）→ 第二幕下·复杂化（25%）→ 第三幕·揭示（25%）

三、章节大纲
   | 章节 | 标题 | 核心事件 | 线索操作 | 嫌疑转移 | 关系翻面 | 张力指数 | 钩子类型 |

四、双时间线设计（故事时间线 vs 叙事时间线）

五、信息阶梯（5层）
   案件表象 → 嫌疑生态 → 关系翻面 → 动机反转 → 情感真相
   规则：每章回答1个问题，打开1-2个更昂贵的新问题

六、钩子地图 + 七、视角与知识对齐计划
```

**中点类型**：假揭示 / 受害者翻转 / 案件升级 / 侦探个人化 / 线索翻转

**结构原则**：承诺原则 / 一层回答两层追问 / 不可逆转 / 场景必须有价值翻转

### 阶段 6：逐章创作

加载参考：[suspense-technique.md](references/suspense-technique.md)

每章输出 `chapters/ch{NNN}.md`，包含：场景信息 → 场景博弈表(表层议题/潜台词战场/试探路径/权力变化/信息操作) → 正文(800-3000字) → 章末钩子 → 创作备注(线索操作/嫌疑变化/关系变化/读者推理状态/钩子类型)

**场景写作原则**（Hitchcock）：悬念≠惊吓 / 戏剧反讽 / 对话=试探+隐藏+换取时间 / 权力必须移动 / 每场景至少推进一项：新线索/排除可能/增加复杂/改变嫌疑/升级风险/揭示秘密

## 质量评估

6维度评分（每项1-10分，满分60分，按子类型加权）：悬念张力 / 线索技术 / 人物一致性 / 节奏控制 / 逻辑严密性 / 文学质量

评级：<30需重写 → 30-39可接受 → 40-49良好 → 50-60优秀

详细评分权重见 [evaluation-rubric.md](references/evaluation-rubric.md)

## 线索一致性审查

任何阶段可执行：线索完整性 → 可见性 → 红鲱鱼状态 → 公平性评分(Knox按子类型过滤) → 双读者验证 → 漏洞检测

## 修改与打磨

四种改稿模式：结构改稿(宏观节奏) / 角色改稿(人物一致性) / 线索改稿(推理逻辑) / 文字改稿(逐章润色)

## 作品导出

全部章节完成后生成：完整版合并文件 / 作品概要 / 线索-章节对照表 / 角色关系图(Mermaid) / 时间线图表

## 工作流路线

```
路线A（本格/密室/叙述诡计 — 诡计优先）：
  初始化 → 主题 → 诡计设计 → 角色 → 结构 → 逐章创作 → 审查 → 修改 → 导出

路线B（社会派/心理悬疑 — 角色优先）：
  初始化 → 主题 → 角色 → 诡计设计 → 结构(简化) → 逐章创作 → 审查 → 修改 → 导出

路线C（惊悚/犯罪程序 — 结构优先）：
  初始化 → 主题 → 结构 → 角色 → 诡计设计 → 逐章创作 → 审查 → 修改 → 导出
```
