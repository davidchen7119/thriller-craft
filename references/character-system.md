# 角色系统 — Character System

> 融合 Robert McKee 角色理论、David Corbett 角色罗盘、John Truby 22步角色设计
> 针对悬疑/推理类型特化的角色设计框架 + 群像关系引擎

---

## 一、悬疑核心角色四元组

每部悬疑作品的最小角色配置：

| 角色功能 | 叙事作用 | 读者关系 |
|----------|----------|----------|
| 侦探/调查者 | 读者的代理人，推动调查 | 认同 |
| 犯人/对手 | 制造谜团，对抗侦探 | 追踪 |
| 受害者 | 案件的起点和情感锚点 | 同情 |
| 迷雾角色 | 提供信息/误导，增加复杂度 | 猜测 |

## 二、侦探/调查者原型

### 七种侦探原型

| 原型 | 代表 | 核心方法 | 人格特质 | 缺陷 |
|------|------|----------|----------|------|
| 理智型 | 福尔摩斯 | 观察+逻辑演绎 | 超然、精确 | 社交障碍、傲慢 |
| 直觉型 | 马普尔 | 人性经验+类比 | 温和、洞察 | 被低估、年龄 |
| 程序型 | 马丁·贝克 | 系统调查+团队 | 严谨、耐心 | 官僚束缚、疲惫 |
| 受困型 | 普通人卷入 | 求生本能+常识 | 脆弱、真实 | 缺乏专业能力 |
| 不端型 | Philip Marlowe | 街头智慧+道德底线 | 愤世嫉俗、正义 | 酗酒、孤独 |
| 局内型 | 嫌疑人视角 | 自证清白+反向推理 | 焦虑、决断 | 不被信任 |
| 创伤型 | 林肯·莱姆 | 以弱胜强+独特视角 | 坚韧、偏执 | 个人创伤限制 |

### 侦探设计检查清单

```
□ 侦探有独特的推理方法论（不是泛泛的"聪明"）
□ 侦探有与案件无关的个人生活/爱好
□ 侦探有一个核心缺陷
□ 侦探的缺陷在案件中被放大/考验
□ 侦探在破案过程中经历内在变化
□ 侦探为什么要管这件事？（动机必须超越"这是我的工作"）
```

### 侦探的个人 Stakes

好的悬疑不仅是"能否破案"，更是"破案对侦探意味着什么"：

```
低 Stakes：丢掉工作/名誉
中 Stakes：某个关心的人受威胁
高 Stakes：侦探自己的生存
最高 Stakes：侦探对自我认知的挑战（"我以为自己是谁"）

理想配置：至少两层 Stakes 叠加
```

## 三、犯人/反派设计

### 犯人的四个维度

```
一、动机（为什么这么做？）

  可信动机谱：
  ├── 本能驱动：恐惧、愤怒、欲望
  ├── 情感驱动：嫉妒、报复、爱（扭曲的）
  ├── 理性驱动：利益、保护秘密、消除威胁
  ├── 信念驱动：正义感（偏执的）、使命感
  └── 结构驱动：制度逼迫、别无选择

  动机深度要求：
  - 表面动机：侦探最先想到的原因
  - 真实动机：更深层的个人原因
  - 根源动机：塑造这个人走向犯罪的生命经历

二、能力（怎么做到的？）

  - 专业技能：医学、化学、工程、IT...
  - 资源掌控：金钱、权力、人脉
  - 心理操控：欺骗、诱导、威胁
  - 机会优势：职位、关系、地理位置

三、伪装（如何不被发现？）

  - 社会面具：在他人眼中是什么人？
  - 日常行为：如何维持正常生活？
  - 应对调查：面对审问时的策略
  - 反侦察：主动消除证据的手段

四、弧光（犯人的变化轨迹）

  - 犯罪前：推动 ta 越过道德红线的过程
  - 犯罪后：心理变化（内疚/麻木/合理化/享受）
  - 被调查时：压力下的行为变化
  - 暴露时：最终面对真相时的反应
```

### 犯人的"同理性"设计

最好的悬疑犯人让读者在得知真相后产生复杂情感：

```
Level 1："这个人是怪物"（纯粹邪恶）— 可接受但缺乏深度
Level 2："这个人做了坏事"（行为邪恶，人格正常）— 标准
Level 3："我理解 ta 为什么这么做"（可理解的动机）— 好
Level 4："如果我是 ta，我可能也会..."（令人不安的共情）— 优秀
Level 5："到底谁才是受害者？"（颠覆善恶二元论）— 社会派巅峰
```

## 四、受害者设计

### 受害者不是道具

```
受害者设计框架：

一、受害者的 N 个面孔
  - 公众形象：社会对 ta 的一般认知
  - 亲友形象：亲近的人眼中的 ta
  - 秘密形象：ta 隐藏的那一面
  - 犯人眼中的 ta：犯人为什么选择 ta

二、受害者的秘密
  每个受害者至少有 1 个与案件相关的秘密
  秘密的揭示是中段推动情节的主要动力

三、受害者的关系网
  受害者与每个主要角色的关系定义
  关系中的冲突点 = 嫌疑来源

四、受害者的"声音"
  即使已死亡，受害者也应该通过以下方式"出现"：
  - 日记/信件/邮件/社交媒体
  - 其他角色的回忆
  - 遗留的物品和空间
  - 生前的录音/视频
```

## 五、嫌疑人矩阵设计

### 嫌疑人配置规则

| 类型 | 数量建议 | 功能 |
|------|---------|------|
| 首要嫌疑人 | 1-2 人 | 第一幕末看起来最可疑，后被排除 |
| 次要嫌疑人 | 2-3 人 | 中段各有可疑时刻 |
| 隐藏嫌疑人 | 1 人 | 真凶，前期看起来最不可能 |
| 红鲱鱼嫌疑人 | 0-1 人 | 纯误导，有独立的秘密 |

### 嫌疑指数曲线

每个嫌疑人的嫌疑程度应随故事发展变化：

```
嫌疑度
 ↑
 │  A╲     ╱C─╲
 │    ╲   ╱    ╲
 │     ╲ ╱  B╱─ ╲___
 │      X   ╱       D──── ← 真凶在最后时刻嫌疑陡升
 │     ╱ ╲_╱
 │    ╱
 └────────────────────→ 章节

A：首要嫌疑人，先高后低
B：次要嫌疑人，中段上升后回落
C：另一嫌疑人，与 A 交替
D：真凶，长期低嫌疑，最后陡升
```

### 每个嫌疑人需要回答

```
□ 动机：ta 为什么可能想这么做？
□ 机会：ta 在案发时在哪里？
□ 能力：ta 有没有做到这件事的条件？
□ 不在场证明：有没有？可靠吗？如何被打破（如果是真凶）？
□ 独立秘密：ta 自己的秘密是什么？（与案件无关但增加嫌疑）
□ 消除方式：如何让读者信服 ta 不是犯人？（如果不是真凶）
```

## 六、配角功能表

| 功能角色 | 叙事作用 | 设计要点 |
|----------|----------|----------|
| 助手/搭档 | 侦探的能力补充、对话对象 | 专长与侦探互补，不能太笨也不能太强 |
| 信息源 | 提供关键信息 | 必须有提供/不提供信息的合理理由 |
| 对立力量 | 阻碍调查 | 不一定是坏人（律师、上级、受害者家属） |
| 道德镜像 | 反映主角的道德选择 | 代表主角可能选择的另一条路 |
| 情感锚点 | 提供主角的情感支持 | 让主角有"可以失去的东西" |
| 社会窗口 | 展示案件的社会背景 | 记者、邻居、社区成员 |

## 七、角色关系图设计

### 必须定义的关系维度

```
角色 A — 角色 B 之间：

1. 表面关系：社会身份上的关系（同事、朋友、家人...）
2. 情感关系：真实的情感连接（信任/恐惧/爱/恨/嫉妒...）
3. 信息关系：A 知道 B 的什么？B 知道 A 的什么？
4. 权力关系：谁在这段关系中有更多权力？
5. 秘密关系：这段关系中隐藏了什么？
```

### 关系网络的悬疑增效

```
秘密链：A 知道 B 的秘密 → B 知道 C 的秘密 → C 知道 A 的秘密
  效果：每个人都在保护自己的同时隐瞒别人

三角关系：A-B 看似亲密 → A-C 有秘密联系 → B-C 互相敌视
  效果：任何一条关系的揭露都重新配置整个网络

代际创伤：上一代的事件影响下一代的关系
  效果：提供历史深度和多层动机
```

## 八、角色罗盘（David Corbett 方法）

### 四维动机模型

每个主要角色（尤其是侦探和犯人）用四个维度定义动机：

```
Lack（匮乏）：角色生命中缺失了什么？
  → 这是伤口，是驱动一切行为的根源

Yearning（渴望）：角色灵魂深处真正需要什么？
  → 这是未被意识到的深层需求

Resistance（抗拒）：什么阻止角色追求真正的渴望？
  → 内在的防御机制、恐惧、信念

Desire（欲望）：角色实际追求的具体目标？
  → 通常是渴望的错误替代品
```

### 动机罗盘的悬疑应用

```
犯人的动机罗盘：
  Lack → 犯罪的心理根源
  Yearning → ta真正想要的（可能是正当的）
  Resistance → 为什么不用合法方式追求
  Desire → 犯罪行为的直接目标

侦探的动机罗盘：
  Lack → 为什么要管这件事
  Yearning → 破案对ta个人意味着什么
  Resistance → 什么可能让ta放弃
  Desire → 表面上的目标（找到真相）

两者的对称性：
  最好的悬疑中，侦探和犯人有同源的 Lack
  但因为不同的 Resistance，走向了不同的 Desire
  → "我们本可以是同一个人" = 最强烈的揭示
```

### Truby 的"伪盟友对手"

```
Fake-Ally Opponent（伪盟友对手）：
  看起来是主角的盟友，实际上是对手。
  或者：看起来是对手，实际上是盟友。

  在悬疑中的应用：
  - 嫌疑人矩阵中必须有至少一个"伪盟友"
  - 这个角色在前半段帮助侦探
  - 后半段揭示ta的真实立场
  - 揭示可以是负面的（ta是帮凶）或正面的（ta在暗中保护侦探）

Attack by Ally（盟友的攻击）：
  主角偏离道德航线时，真正的盟友会指出来。
  在悬疑中 = Watson 角色质疑侦探的方法/判断。
  这个时刻让读者重新评估侦探的可靠性。
```

## 九、群像关系引擎

> 适用于封闭圈层悬疑：人物不是功能点，而是持续博弈的系统
> 核心问题不是"谁做了"，而是"谁在维持秩序，谁在争夺秩序，谁会因为真相失去一切"

### 启用条件

```
当故事满足以下任意 2 项时，不要只写"角色设定"，要改用"关系引擎"：
  □ 人物池相对固定（夜场、家族、公司、学校、剧团、社区、警局）
  □ 角色之间有长期债务/依附/竞争，而非一次性相遇
  □ 案件真相会重写人物关系，而不只是揭露凶手
  □ 每一集/章都需要结尾钩子来维持追看
  □ 读者的兴趣一半来自"人和人之间会怎么翻脸"

关系引擎的目标：
  1. 让每个角色拥有独立逻辑闭环
  2. 让每场戏都同时推进信息、关系、危险
  3. 让真相揭示具备情感后坐力
```

## 十、关键角色 Agent 卡

对主角、反派、核心嫌疑人都填写同一张卡，避免角色只在自己的章节里成立。

| 字段 | 问题 | 例子 |
|------|------|------|
| 社会身份 | 这个角色在系统里扮演什么位置？ | 妈妈桑 / 红牌 / 刑警 / 家族长女 |
| 面具人格 | ta 希望别人相信自己是什么人？ | 温柔体贴 / 可靠能干 / 无害 / 忠诚 |
| 真实意图 | ta 真正想保住或夺走什么？ | 控制权 / 生存空间 / 爱 / 资源 / 清白 |
| 核心恐惧 | 什么暴露会让 ta 失控？ | 被抛弃 / 被取代 / 被看穿 / 被剥夺地位 |
| 当下欲望 | 这场戏里 ta 最想拿到什么？ | 一句承诺 / 一个答案 / 一次拖延 |
| 筹码 | ta 现在最有力的资源是什么？ | 钱 / 名声 / 性吸引 / 暴力 / 信息 / 关系 |
| 依赖 | ta 离不开谁的认可、保护或沉默？ | 上司 / 伴侣 / 母亲 / 舆论 / 关键证人 |
| 不能暴露的秘密 | 什么一旦见光就会改写全局？ | 旧案关联 / 真实身份 / 债务 / 亲密关系 |
| 破裂触发器 | 什么事件会让 ta 立刻换边或开战？ | 被公开羞辱 / 发现被背叛 / 失去资源入口 |

最小检查：
  □ 面具人格和真实意图至少在一个维度上冲突
  □ 筹码不是抽象词，必须能在场景里被使用
  □ 依赖必须指向具体人物或具体系统

## 十一、关系边设计

对每一条关键关系边，至少写出下面 5 项：

| 关系边 | 公开关系 | 隐性张力 | 资源流 | 知识优势 | 翻面条件 |
|--------|----------|----------|--------|----------|----------|
| A ↔ B | 朋友/上下级/恋人/竞争者 | 嫉妒/欠债/欲望/羞耻 | 谁给谁钱、保护、机会 | 谁知道对方的哪条秘密 | 什么事件会让关系瞬间改写 |

高价值关系边通常同时具备三件事：感情、利益、信息不对称。

如果一条关系边只有感情，没有利益或秘密，它很难持续发电。

## 十二、封闭系统设计

```
一个好用的悬疑场域，通常有三层秩序：

  表层秩序：这里看起来是怎么运转的
  地下秩序：真正决定资源分配的是谁
  崩塌点：什么真相会让整个系统失衡
```

设计时至少回答：

| 问题 | 示例 |
|------|------|
| 谁在维持表面秩序？ | 店长 / 长姐 / 主持人 / 队长 |
| 谁是秩序的实际受益者？ | 老板 / 资方 / 核心红牌 / 老资格成员 |
| 谁最想改变秩序？ | 新人 / 边缘人 / 被亏待者 / 被替代者 |
| 谁看起来有权力，其实最脆弱？ | 温柔控制者 / 名义领导 / 人气人物 |
| 谁看起来最弱，实际握着开关？ | 会计 / 助理 / 司机 / 被忽视的见证人 |

不要只写"这是一个封闭空间"，要写清楚：为什么他们不能轻易离开、为什么不能轻易翻脸、为什么必须继续同桌吃饭、一起上班、继续互相依赖。

## 十三、信息阶梯

悬疑群像最容易坏在信息一次性讲完。改用阶梯式释放：

```
Layer 1  案件表象 — 谁死了？出了什么事？谁最可疑？
Layer 2  嫌疑生态 — 为什么每个人都像有动机？
Layer 3  关系翻面 — 原来 A 和 B 不是我们以为的关系。
Layer 4  动机反转 — 原来 ta 争的不是钱/爱/清白，而是更深的东西。
Layer 5  情感真相 — 真相公开后，读者终于理解这群人为什么会走到这一步。
```

章节规则：
  □ 每章至少推进一层，或者让旧层获得新含义
  □ 每章最好回答 1 个问题，并打开 1-2 个更贵的问题
  □ 中点前后至少发生一次关系翻面，而不是只补线索

## 十四、对话交锋

### 场景对话决斗表

写多人对话前，先填这个最小表：

| 角色 | 表层议题 | 真正想拿到的东西 | 不能直接说的内容 | 试探对象 | 失手风险 |
|------|----------|------------------|------------------|----------|----------|
| A | 讨论案发时间 | 套出 B 的不在场证明漏洞 | ta 其实去过现场 | B | 暴露自己知道太多 |

对话四原则：
  1. 每句话都至少服务于一种动作：试探 / 隐藏 / 拖延 / 交换筹码
  2. 回答不要只关闭问题，要改写问题
  3. 让角色根据自身筹码说话，而不是共享作者口吻
  4. 至少一次让权力位置发生改变

## 十五、章末钩子设计

### 五种高价值钩子

| 类型 | 作用 | 例子 |
|------|------|------|
| 答案型 | 给读者一个局部满足，但揭开更大的洞 | 证实 A 不是真凶，但他在替谁挡 |
| 威胁型 | 让人物下一章必须行动 | 有人发现主角在撒谎 |
| 关系翻面型 | 改写联盟和站队 | 原来最亲密的两个人早已互相算计 |
| 自我怀疑型 | 让 POV 自身失去稳定性 | 主角开始怀疑自己的记忆 |
| 决策悬崖型 | 把角色推到必须选边的时刻 | 如果现在报警，就会毁掉另一个人 |

钩子检查：
  □ 钩子改变了至少一个角色的处境
  □ 钩子不是单纯"新信息"，而是带代价的信息
  □ 钩子与下一章的场景目标直接相连

## 十六、常见失败模式与修复

| 失败模式 | 表现 | 修复方式 |
|----------|------|----------|
| 角色是立场标签 | 角色只有"强势/温柔/坏人/受害者" | 补齐 Agent 卡中的面具、意图、筹码、依赖 |
| 关系是静态背景 | 关系只在设定里存在，不在场景里变化 | 为每条关键关系定义翻面条件 |
| 对话只有信息没有风险 | 说完了，但谁都没失去或得到什么 | 重写场景博弈表，让对白承担试探和交换 |
| 钩子只有噱头 | 结尾很炸，但下一章接不上 | 让钩子直接改变下一章的行动目标 |
| 真相只有逻辑没有后坐力 | 揭示成立，但不痛 | 回到 Layer 5，补上情感真相 |

---

## 十七、最小执行协议

如果项目是社会派、心理悬疑或连续剧化群像，至少执行以下四步：

1. `/start` 时明确 `socialArena` 与 `relationshipEngine`
2. `/characters` 时完成关键角色 Agent 卡和关系边表
3. `/structure` 时写出信息阶梯与钩子地图
4. `/scene` 时先写场景博弈表，再写正文

少了这四步，故事很容易退化成"有人死了，大家轮流说话"。
